title: Storage Service
description: AsyncStorage wrapper functions for Stoic Calendar data persistence
---

The Storage Service provides a comprehensive AsyncStorage wrapper for all data persistence in Stoic Calendar. It handles timelines, active timeline state, and app settings with a Firebase-ready structure.

## Timeline Operations

### loadTimelines

Load all timelines from storage.

```typescript
export async function loadTimelines(): Promise<Timeline[]>
```

<ResponseField name="return" type="Promise<Timeline[]>">
  Array of timelines (empty array if none exist)
</ResponseField>

**Example:**

```typescript
import { loadTimelines } from '@/services/storage';

const timelines = await loadTimelines();
console.log(`Found ${timelines.length} timelines`);
```

---

### saveTimeline

Save a new timeline or update an existing one.

```typescript
export async function saveTimeline(timeline: Timeline): Promise<void>
```

<ParamField path="timeline" type="Timeline" required>
  Timeline to save
</ParamField>

**Example:**

```typescript
import { saveTimeline } from '@/services/storage';
import { createTimeline } from '@/services/timeline-calculator';
import { TimelineType } from '@/types/timeline';

const newTimeline = createTimeline(TimelineType.YEAR);
await saveTimeline(newTimeline);
```

---

### deleteTimeline

Delete a timeline by ID. If the deleted timeline was active, it clears the active timeline.

```typescript
export async function deleteTimeline(id: string): Promise<void>
```

<ParamField path="id" type="string" required>
  Timeline ID to delete
</ParamField>

**Example:**

```typescript
import { deleteTimeline } from '@/services/storage';

await deleteTimeline('timeline_1234567890_abc123');
```

---

### getTimelineById

Get a timeline by ID.

```typescript
export async function getTimelineById(id: string): Promise<Timeline | null>
```

<ParamField path="id" type="string" required>
  Timeline ID
</ParamField>

<ResponseField name="return" type="Promise<Timeline | null>">

**Example:**

```typescript

}
```

## Storage Keys

The service uses the following internal storage keys:

- `@stoic_calendar:timelines` - All timelines array
- `@stoic_calendar:settings` - App settings object
- `@stoic_calendar:active_timeline_id` - Active timeline ID
