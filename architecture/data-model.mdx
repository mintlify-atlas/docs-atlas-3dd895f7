## Overview

Stoic Calendar uses a Firebase-ready data model with TypeScript interfaces that define the structure for timelines and application settings. All dates are stored in ISO 8601 format for consistency and cloud sync readiness.

## Timeline Interface

The `Timeline` interface is the core data model for all timeline types in the app.

```typescript
interface Timeline {
  /**
   * Unique identifier (UUID v4)
   */
  id: string;

  /**
   * Timeline type: year, month, week, or custom
   */
  type: TimelineType;

  /**
   * Display title
   * Examples: "2026", "This Week", "Project X"
   */
  title: string;

  /**
   * Start date (ISO 8601 format)
   * Examples: "2026-01-01T00:00:00.000Z"
   */
  startDate: string;

  /**
   * End date (ISO 8601 format)
   * Examples: "2026-12-31T23:59:59.999Z"
   */
  endDate: string;

  /**
   * Creation timestamp (ISO 8601 format)
   */
  createdAt: string;

  /**
   * Whether this timeline is currently displayed on the Home screen
   * Only one timeline should have isActive = true at a time
   */
  isActive: boolean;

  /**
   * Widget configuration (reserved for future iOS widget integration)
   */
  widgetPreferences?: WidgetPreferences;
}
```

### Timeline Types

The `TimelineType` enum defines four categories of timelines:

```typescript
enum TimelineType {
  YEAR = 'year',
  MONTH = 'month',
  WEEK = 'week',
  CUSTOM = 'custom',
}
```

**Source:** `types/timeline.ts:5-10`

#### Type Behaviors

- **YEAR**: Fixed year timeline (e.g., "2026")
- **MONTH**: Fixed month timeline (e.g., "January 2026")
- **WEEK**: Auto-updating current week (Monday-Sunday)
- **CUSTOM**: User-defined date range with custom title

## Active Timeline Pattern

Only ONE timeline can have `isActive: true` at a time. The active timeline is displayed on the Home screen.

### Usage Example

```typescript
import { setActiveTimeline, getActiveTimeline } from '@/services/storage';
import { createTimeline } from '@/services/timeline-calculator';
import { TimelineType } from '@/types/timeline';

// Create a new year timeline
const newTimeline = createTimeline(TimelineType.YEAR, {
  year: 2026,
  isActive: false,
});

// Save and set as active
await saveTimeline(newTimeline);
await setActiveTimeline(newTimeline.id); // Automatically updates all timeline flags

// Retrieve active timeline
const activeTimeline = await getActiveTimeline();
```

## Firebase-Ready Structure

All data structures are designed for seamless Firebase/Firestore migration:

- **ISO 8601 dates**: All dates stored as strings in ISO 8601 format
- **Flat structure**: No nested objects that can't serialize
- **String IDs**: UUIDs stored as strings for cross-platform compatibility
- **No functions**: Pure data objects with no methods
