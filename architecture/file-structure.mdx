Stoic Calendar follows Expo Router's conventions with a clear separation of concerns across directories.

## Root Structure

```
stoic-calendar/
├── app/                    # Expo Router pages (file-based routing)
├── assets/                 # Images, fonts, and static resources
├── components/             # Reusable React components
├── constants/              # App-wide constants
├── hooks/                  # Custom React hooks
├── modules/                # Native modules
├── scripts/                # Build and utility scripts
├── services/               # Business logic and data services
├── targets/                # iOS widget targets
├── types/                  # TypeScript type definitions
├── utils/                  # Utility functions
├── app.json                # Expo configuration
├── package.json            # Dependencies
└── tsconfig.json           # TypeScript configuration
```

## app/ - Routing & Screens

File-based routing via Expo Router. Each file becomes a route.

```
app/
├── _layout.tsx             # Root layout with theme provider
├── index.tsx               # Entry point (redirects to /home)
├── home.tsx                # Home screen - active timeline display
├── settings.tsx            # Settings screen
├── paywall.tsx             # RevenueCat subscription paywall
├── customer-center.tsx     # RevenueCat customer center
└── CLAUDE.md               # AI assistant instructions for this directory
```

**Key files:**

### app/_layout.tsx

Root layout that wraps all screens:
- Theme provider setup
- Font loading
- Navigation container
- Safe area context

### app/home.tsx

Main timeline display screen:
- Renders active timeline's StoicGrid
- Auto-updates week timelines
- Timeline dropdown for switching
- Day count and progress display

See: `app/home.tsx`

### app/settings.tsx

User preferences:
- Theme mode (system/light/dark)
- Grid color theme selection
- Subscription status
- Links to customer center

See: `app/settings.tsx`

**Key services:**

### services/storage.ts

Centralized data persistence:

```typescript
// Load all timelines
export async function loadTimelines(): Promise<Timeline[]>

// Save a timeline
export async function saveTimeline(timeline: Timeline): Promise<void>

// Delete a timeline
```

See: `services/storage.ts`

### services/timeline-calculator.ts

**Key types:**

```typescript
export enum TimelineType {
  YEAR = 'year',
  MONTH = 'month',
  WEEK = 'week',
  CUSTOM = 'custom',
}

export interface Timeline {
  id: string;
  type: TimelineType;
  title: string;
  startDate: string;        // ISO 8601
  endDate: string;          // ISO 8601
  createdAt: string;        // ISO 8601
  isActive: boolean;
  widgetPreferences?: WidgetPreferences;
}

export interface AppSettings {
  themeMode: ThemeMode;
  gridColorTheme: GridColorTheme;
}

export type ThemeMode = 'system' | 'light' | 'dark';
export type GridColorTheme = 'classic' | 'forest' | 'sunset' | 'monochrome';
```

See: `types/timeline.ts`

**app-groups module:**

Enables data sharing between the main app and iOS widgets:

```typescript
import AppGroups from 'app-groups';

// Write to shared container
await AppGroups.setItem(
  'group.com.stoiccalendar.shared',
  'activeTimeline',
  JSON.stringify(timeline)
);

// Read from shared container
const data = await AppGroups.getItem(
  'group.com.stoiccalendar.shared',
  'activeTimeline'
);
```

## Path Aliases

The project uses `@/` as root alias:

```typescript
import { Timeline } from '@/types/timeline';
import { loadTimelines } from '@/services/storage';
import { StoicGrid } from '@/components/stoic-grid';
import { getDaysPassed } from '@/utils/date-helpers';
```

Configured in `tsconfig.json`:

```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```
