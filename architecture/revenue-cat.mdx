The RevenueCat Service handles all subscription management, purchase flows, and Pro feature gating using the RevenueCat SDK.

## Overview

**Status**: MVP (Active)

**File**: `services/revenue-cat-service.ts`

**SDK**: `react-native-purchases` (RevenueCat)

## Configuration

### API Key

Loaded from environment variable:

```typescript
const REVENUECAT_API_KEY = process.env.EXPO_PUBLIC_REVENUECAT_API_KEY || '';
```

Set in `.env`:

```bash
EXPO_PUBLIC_REVENUECAT_API_KEY=your_key_here
```

**Important**: Never commit API keys to version control.

## Core Functions

### isPro()

Check if user has active Pro subscription:

```typescript
export async function isPro(): Promise<boolean>
```

**Usage**:

```typescript
import { isPro } from '@/services/revenue-cat-service';

const hasPro = await isPro();

if (hasPro) {
  // Show Pro features
} else {
  // Show paywall
}
```

**Behavior**:
- Fetches customer info from RevenueCat
- Checks for active "Memento Calendar Pro" entitlement
- Returns `false` on error (fail-safe)

### getCustomerInfo()

Get full customer purchase and subscription info:

```typescript
export async function getCustomerInfo(): Promise<CustomerInfo>
```

**Usage**:

```typescript
import { getCustomerInfo } from '@/services/revenue-cat-service';

const info = await getCustomerInfo();
console.log('Active subscriptions:', info.activeSubscriptions);
console.log('Entitlements:', Object.keys(info.entitlements.active));
```

**Returns**: RevenueCat `CustomerInfo` object with:
- `entitlements.active` - Active entitlements
- `activeSubscriptions` - Active subscription product IDs
- `allPurchasedProductIdentifiers` - All purchased products
- `latestExpirationDate` - Latest subscription expiration

### getOfferings()

Get available subscription offerings:

```typescript
export async function getOfferings(): Promise<PurchasesOffering | null>
```

**Usage**:

```typescript
import { getOfferings } from '@/services/revenue-cat-service';

const offering = await getOfferings();

if (offering) {
  const monthly = offering.monthly;
  const yearly = offering.annual;
  
  console.log('Monthly:', monthly?.product.priceString);
  console.log('Yearly:', yearly?.product.priceString);

```

**Returns**: Current offering with:
- `monthly` - Monthly subscription package
- `annual` - Yearly subscription package
- `availablePackages` - All available packages

### purchasePackage()

Purchase a subscription package:

```typescript
export async function purchasePackage(
  packageToPurchase: PurchasesPackage
): Promise<{ customerInfo: CustomerInfo; userCancelled: boolean }>
```

**Usage**:

```typescript
import { getOfferings, purchasePackage } from '@/services/revenue-cat-service';

const offering = await getOfferings();
if (!offering?.monthly) return;

try {
  const { customerInfo, userCancelled } = await purchasePackage(offering.monthly);
  
  if (userCancelled) {
    console.log('User cancelled purchase');
  } else {
    console.log('Purchase successful!');
    // Navigate to Pro features
  }
} catch (error) {
  console.error('Purchase failed:', error);
}
```

**Behavior**:
- Handles App Store/Play Store purchase flow
- Returns `userCancelled: true` if user cancels
- Throws error on purchase failure
- **Automatically syncs Pro status to widgets**

## Feature Gating Example

Complete example of Pro feature gating:

```typescript
import { isPro, FREE_TIER_LIMITS } from '@/services/revenue-cat-service';
import { loadTimelines } from '@/services/storage';

async function canCreateTimeline(): Promise<boolean> {
  const hasPro = await isPro();
  
  if (hasPro) {
    return true; // Unlimited for Pro
  }
  
  // Check free tier limit
  const timelines = await loadTimelines();
  return timelines.length < FREE_TIER_LIMITS.MAX_TIMELINES;
}

async function handleCreateTimeline() {
  const canCreate = await canCreateTimeline();
  
  if (!canCreate) {
    // Show paywall
    navigation.navigate('Paywall');
    return;
  }
  
  // Create timeline
  const newTimeline = createTimeline(/* ... */);
  await saveTimeline(newTimeline);
}
```

## Error Handling

All functions handle errors gracefully:

```typescript
try {
  const hasPro = await isPro();
} catch (error) {
  console.error('Error checking pro status:', error);
  // Fail-safe: assume not Pro on error
}
```

Key error scenarios:
- Not initialized: Throws error
- Network failure: Throws error
- User cancellation: Returns `userCancelled: true`
- Store error: Throws error

## Best Practices

1. **Initialize early**: Call `initializeRevenueCat()` in app root
2. **Check Pro status**: Always check `isPro()` before gating features
3. **Handle cancellation**: Check `userCancelled` in purchase flow
4. **Restore purchases**: Provide restore button in settings
5. **Mock for development**: Use mock mode for UI development
6. **Error handling**: Always wrap calls in try-catch
7. **User feedback**: Show loading states during purchase/restore
