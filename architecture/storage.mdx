## Overview

The Storage Service (`services/storage.ts`) provides a centralized AsyncStorage wrapper for all data persistence operations in Stoic Calendar. It manages timelines, app settings, and active timeline state with Firebase-ready data structures.

## Storage Keys

All data is stored under namespaced keys:

```typescript
const STORAGE_KEYS = {
  TIMELINES: '@stoic_calendar:timelines',
  SETTINGS: '@stoic_calendar:settings',
  ACTIVE_TIMELINE_ID: '@stoic_calendar:active_timeline_id',
} as const;
```

**Source:** `services/storage.ts:25-29`

## Timeline Operations

### Load All Timelines

Retrieve all saved timelines from AsyncStorage:

```typescript
export async function loadTimelines(): Promise<Timeline[]>
```

**Returns:** Array of timelines (empty array if none exist)

**Example:**
```typescript
import { loadTimelines } from '@/services/storage';

const timelines = await loadTimelines();
console.log(`Found ${timelines.length} timelines`);
```

**Source:** `services/storage.ts:47-58`

### Save Timeline

Save a new timeline or update an existing one:

```typescript
export async function saveTimeline(timeline: Timeline): Promise<void>
```

**Behavior:**
- If timeline with matching `id` exists, it updates the existing timeline
- If timeline is new, it appends to the array
- Automatically serializes to JSON for AsyncStorage

**Example:**
```typescript
import { saveTimeline } from '@/services/storage';
import { createTimeline } from '@/services/timeline-calculator';
import { TimelineType } from '@/types/timeline';

const newTimeline = createTimeline(TimelineType.YEAR, { year: 2026 });
await saveTimeline(newTimeline);
```

**Source:** `services/storage.ts:64-84`

### Delete Timeline

Delete a timeline by ID:

```typescript
export async function deleteTimeline(id: string): Promise<void>
```

**Behavior:**
- Removes timeline from storage
- If deleted timeline was active, clears active timeline reference

**Example:**
```typescript
import { deleteTimeline } from '@/services/storage';

await deleteTimeline('timeline_1709740800000_abc123xyz');
```

**Source:** `services/storage.ts:90-106`

## Active Timeline Operations

### Set Active Timeline

Set a timeline as active (displayed on Home screen):

```typescript
export async function setActiveTimeline(id: string): Promise<void>
```

**Behavior:**
- Updates `ACTIVE_TIMELINE_ID` in storage
- Sets `isActive: true` on the specified timeline
- Sets `isActive: false` on all other timelines
- Syncs data to iOS/Android widgets (non-fatal if fails)

**Example:**
```typescript
import { setActiveTimeline } from '@/services/storage';

await setActiveTimeline('timeline_1709740800000_abc123xyz');
```

**Source:** `services/storage.ts:146-170`

## Widget Sync Integration

The storage service automatically syncs data to iOS/Android widgets using a lazy-loaded widget service:

```typescript
const syncWidgetData = async (type: 'timeline' | 'settings') => {
  const { syncActiveTimelineToWidget, syncSettingsToWidget } = 
    await import('@/services/widget-data-service');
  if (type === 'timeline') {
    await syncActiveTimelineToWidget();
  } else {
    await syncSettingsToWidget();
  }
};
```

**Note:** Widget sync failures are non-fatal and logged as warnings.

**Source:** `services/storage.ts:13-20`

## Error Handling

All storage operations include try-catch blocks and error logging:

```typescript
try {
  const data = await AsyncStorage.getItem(STORAGE_KEYS.TIMELINES);
  // ... operation logic
} catch (error) {
  console.error('Error loading timelines:', error);
  return []; // Safe fallback
}
```

Operations that modify data throw errors to the caller, while read operations return safe defaults.

## Firebase Migration Path

The storage layer is structured for easy Firebase migration:

1. **Replace AsyncStorage with Firestore**:
   - `loadTimelines()` → `collection('timelines').get()`
   - `saveTimeline()` → `doc('timelines/{id}').set()`
   - `deleteTimeline()` → `doc('timelines/{id}').delete()`

2. **No data transformation needed**: All data is already in Firebase-compatible format

3. **Add real-time listeners**: Replace async loads with `onSnapshot()` listeners
