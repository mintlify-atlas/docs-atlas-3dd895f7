## Architecture Considerations

### Data Migration Strategy

**Local to Cloud Migration:**

```typescript
export async function migrateToCloud(): Promise<void> {
  // 1. Check if user is authenticated
  const user = getCurrentUser();
  if (!user) throw new Error('User not authenticated');
  
  // 2. Load all local timelines
  const localTimelines = await loadTimelines();
  
  // 3. Check for existing cloud data
  const cloudTimelines = await fetchCloudTimelines(user.uid);
  
  // 4. Merge local and cloud (prefer cloud if conflict)
  const merged = mergeTimelines(localTimelines, cloudTimelines);
  
  // 5. Upload merged data to Firestore
  await uploadAllTimelines(merged);
  
  // 6. Enable cloud sync
  await enableCloudSync();
}
```

### Offline-First Design

The app must work offline:

```typescript
// Always read from local cache first
const timelines = await loadTimelines();

// Sync with cloud in background
if (isOnline()) {
  syncTimelines().catch(error => {
    console.error('Background sync failed:', error);
    // Continue working with local data
  });
}
```

### Conflict Resolution

For simultaneous edits:

```typescript
function resolveConflict(local: Timeline, cloud: Timeline): Timeline {
  // Strategy: Last Write Wins (based on updatedAt timestamp)
  const localTime = new Date(local.updatedAt || local.createdAt).getTime();
  const cloudTime = new Date(cloud.updatedAt || cloud.createdAt).getTime();
  
  return cloudTime > localTime ? cloud : local;
}
```
